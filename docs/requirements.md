# 项目需求文档

## 项目名称

军棋棋子图像识别系统 (Junqi Piece Recognition System)

## 项目目标

开发一套基于 C++ 和 OpenCV 的图像处理算法，能够自动识别照片中的军棋棋子颜色和文字内容。

## 功能需求

### F1: 图像预处理
- 输入：彩色照片（JPEG/PNG）
- 输出：去噪增强后的灰度图和彩色图
- 要求：保留棋子字符笔画细节，抑制反光

### F2: 棋子检测
- 输入：预处理后的图像
- 输出：两枚棋子的 ROI 区域，区分左右
- 要求：检测率 >95%

### F3: 颜色分类
- 输入：棋子 ROI 彩色图
- 输出：红色 / 黑色
- 要求：准确率 >98%，能应对红色棋子反光

### F4: 字符识别
- 输入：棋子 ROI 灰度图
- 输出：棋子中文名称（12 种之一）
- 要求：准确率 >95%

### F5: 模板采集
- 功能：从标准棋子照片中提取字符模板
- 输出：64×64 二值 PNG 模板文件

### F6: 准确率评测
- 功能：批量测试并输出混淆矩阵和准确率报告

## 非功能需求

- 平台：Linux (x86_64)，后续可移植 ARM
- 语言：C++17
- 依赖：OpenCV 4.x
- 性能：PC 上单张处理 <500ms
- 输入规格：照片中包含左右两枚棋子，一红一黑
